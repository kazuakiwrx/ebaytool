# 仕入先 共通ルール（在庫補充・セラーリサーチ 両タスク共通）

このファイルが仕入先選定・価格・利益率の唯一の基準。変更はここだけ直してpushすれば両タスクに反映される。
（このファイルは GitHub Pages の公開対象外。自動化のみが clone して読む）

## 仕入先サイト
- **【暫定・スピード優先】当面、仕入先は「メルカリ」だけに限定する。** ヤフオク／PayPayフリマは一時停止（1件あたりの調査時間を短縮するため）。本ファイル下部や各タスクのプロンプトに「3サイト」「ヤフオク」「PayPay」とあっても、**この暫定指示を最優先**する。元に戻す時はこの行を消す＝下記の平常ルール（3サイト）に戻る。
- （平常時のルール）使うのは メルカリ／ヤフオク（即決のみ・オークション除外）／PayPayフリマ の3つ。それ以外（ラクマ・楽天・Amazon・ハードオフ・駿河屋等）は使わない。
- 即決在庫のみ。複数候補は **送料込み価格の安い順** に並べ、原則1位（最安）を採用。

## 検索（仕入先の見つけ方）— recall重視（スキップを減らす）
**「見つからずスキップ」を出す前に、必ず以下を尽くすこと。** eBayタイトルをそのまま日本のフリマで検索すると当たらない（英語＋出品文句のため）。日本語＋型番で探す。
- **型番・品番を最優先**（言語非依存で最も精度が高い）。例：DW-5600STT-1JR / TD-002-J / RG 1/144 / LEGO 10233 等。型番があれば**まず型番単体で各サイト検索**。
- **日本語の正式名称に変換して検索**：英語の商品名・ブランド・キャラ名・シリーズ名・セット名を日本語へ（例 "Godzilla Soft Vinyl"→「ゴジラ ソフビ」、"Char Aznable"→「シャア・アズナブル」、"Tallgeese"→「トールギス」）。
- **eBay特有の不要語を除去**してから検索：From Japan / F/S / Free Shipping / Limited to N units / NEW / Brand New / Pre-owned / JP / Japan import / セラーの宣伝文句・数量表記など。
- **複数クエリを試す**（1つで出なくても即スキップしない）：(1)型番 (2)日本語商品名 (3)ブランド+キャラ/シリーズ (4)短いコア語のみ。語を減らして網を広げる→候補が出たら商品ページで同一性を確認。
- **3サイト（メルカリ／ヤフオク即決／PayPay）すべてで検索**。各サイト、上位だけでなくスクロールして複数件を確認。
  - メルカリ在庫あり検索URL例：https://jp.mercari.com/search?keyword=<URLエンコード>&status=on_sale
  - ヤフオクは検索後「即決（定額）」で絞る（オークションのみは対象外）。PayPayフリマも購入可能なもの。
- **売り切れ／取得失敗（買値が空）を「在庫なし」と即断しない**：同一商品の別の出品を試す、ID⇄URL再試行、別サイトへ回す（フォールバック）。
- **マッチ判定**：型番・構成・付属が一致する同一商品ならOK。コンディションはeBayのNew/Usedクラスに合わせる（細かいグレード差で弾きすぎない）。「ケースのみ/空箱/部品のみ/単品」等の明確に別物だけ除外。画像（eBay掲載写真と仕入先写真）も同一性チェックの補助に使う。
- 上記をすべて尽くしても見つからない時だけ status:"skipped"（note に試した型番/日本語キーワードを残す）。

## コンディション
- 新仕入先の商品は **元(eBay)のコンディションと同じもの** を選ぶ。
- 元の仕入先表記と eBay商品ページ(#conditionid)が食い違う場合は **eBay商品ページを正** とし、それに一致するものを選ぶ。#conditionid は変更しない。
- Used は新仕入先ページの状態（例「目立った傷や汚れなし」）を英訳して Condition Description に設定。New は商品説明変更スキップ。

## 鮮度
- **掲載6カ月以上前** の仕入先商品は対象外（各サイトの掲載日/経過表示、ヤフオクは開始日で判断）。

## 送料・原価
- **送料込みで比較**。送料が発生する場合は送料も仕入価格に含める。
- **発送先は岐阜県**。MONODAZ予想利益計算は #buyprice（商品代）+ #buyshippingprice（送料）を原価に入れる。送料が0のままで実際は送料がかかる場合は送料額を入力。
- **【出品時の送料は最低¥3,500】出品時の送料（#outshippingprice ＝ MONODAZの「送料取得」値、利益計算で差し引かれる国際送料）は最低¥3,500とする。** 送料取得や初期値が¥3,500未満（例：MONODAZで¥1,500と出ても）の場合は、#outshippingprice を **¥3,500に上書きしてから**「予想利益計算」を実行して価格を決める（実送料を保守的に見込むため）。¥3,500以上ならそのまま使う。

## 仕入先取得フォールバック
- 「仕入先取得」しても買値(#buyprice)が空のままなら失敗とみなす。
- (a) #itemlist-supplierval を **フルURL⇄IDに切り替えて1回再試行** → (b) それでも空なら **次に安い有効候補へ切替えて再取得**。
- 全候補で取得不可ならスキップ（理由を記録）。※PayPayフリマはフルURLで通らないことがあり、ID指定や別候補で再取得。

## 画像（写真）の入替（在庫補充で必ず実施）
- 在庫補充で出品（変更する）前に、**出品写真を新しい仕入先の写真に必ず入れ替える**（古い/売り切れ仕入先の写真のまま残さない）。「仕入先取得」で自動入替されることが多いが、**入れ替わったか必ず確認**（画像src が mercdn 等の仕入先ドメインになっているか）。入れ替わっていなければ #imgchange に新仕入先の画像URL（または仕入先商品URL）を入れて「画像入替」ボタンで手動入替する。
- **【例外・最優先】現在の既存写真に「yui_chan_store」のロゴ／ウォーターマークが入っている場合は、写真を入れ替えない（既存写真を維持）。** 出品前に現在の写真を確認（必要ならスクリーンショット）し、yui_chan_storeロゴ有り→入替えない／ロゴ無し→新仕入先写真に入替える。
- 仕入先側の写真にロゴ/ウォーターマークが入っている場合は、その写真は使わずロゴ無しの別写真・別出品を選ぶ。

## 高額品
- 仕入価格（送料込み = #buyprice+#buyshippingprice）が **¥100,000以上** の商品は自動出品しない（高額のため要承認/人手確認に回す。タスク側の運用に従う）。

## 利益率・価格（タスクで異なる）
- **在庫補充（アラート版 monodaz-auto-listing ／ 出品リストQty0版 monodaz-restock-by-quantity 共通）**：出品価格は **利益率12〜14%の範囲に収める（狙いは中央の約13%）**ように調整（利益率 = 利益計 ÷ (Price × 155.8円)）。逆算（C=a·P1−profit1, a≒134.8 → P目標=C/(a−0.13×155.8)≒C/114.6（狙い13%の初期値））は初期値の目安にすぎない。**最終は必ずMONODAZの「予想利益計算」の実値で確認し、利益率が12〜14%の範囲に入るまで価格を上下して繰り返す（12%未満なら値上げ・14%超なら値下げ、回数制限なし）。利益率が目標未満／マイナスのままでは絶対に保存（変更する）しない。** 妥当な価格で12〜14%に収められない（仕入が高すぎる等）場合は、損失を出して出品せず、見送って要承認(pending)に回すかスキップする。※各タスクのプロンプトに「r の±1%以内」「10%」等とあっても、この12〜14%（狙い13%）を最優先する。
- **セラーリサーチ(monodaz-hourly-10listings)**：出品価格は **「検索ebay price（リサーチ最安値）− $0.5」を基準**。最低利益率（現状5%）を下回る場合のみ満たすまで価格を上げ、下回らなければ最安値−$0.5のまま出品（無理に上げない）。

## 出品対象・除外（カテゴリ/ブランド）
- **トレーディングカード（ポケモンカード等）**：鑑定（グレーディング：PSA/BGS等）付きに限り出品OK。鑑定なしの生カードは対象外（スキップ）。eBay側・仕入先側ともに鑑定品であることを確認する。
- **コミック（本・漫画）**：出品禁止（対象外＝スキップ）。
- **Gucci／Louis Vuitton（ルイヴィトン）**：出品対象（OK）。

### 価格差ルール（出品／要承認／スキップ）
0. **先に除外/高額チェック**：コミック禁止→スキップ。トレカは鑑定付きでなければスキップ。新仕入価格（送料込み）¥100,000以上なら要承認(pending)。
1. **元の仕入価格が取得できる場合のみ** pct=(元−新)/元×100。pct≥10（新が10%以上安い）→出品しない・pending記録。pct<10→自動出品。
2. **元の仕入価格が不明**な場合は pct省略し、有効な仕入先（共通ルール満たす即決・¥100,000未満・除外でない）があれば**目標利益率（12〜14%・狙い13%）で自動出品**。¥100,000以上なら要承認。有効仕入先なしならスキップ。

### 出品フロー詳細（共通）
モーダル要素：仕入先=#itemlist-suppliername、URL=#itemlist-supplierval、取得=#itemlist-modal-getsupply-btn、Price=#price、Quantity=#quantity、SKU=#customlabel、Condition=#conditionid、商品代=#buyprice、送料=#buyshippingprice、出品時送料=#outshippingprice、利益計算=#expectprofitbtn、保存=#item-edit-form-btn（「変更する」）、画像URL入力=#imgchange、画像入替ボタン=「画像入替」。
1. `window.confirm=()=>true`（画像入替ダイアログ自動許可）。
2. **【まとめ実行＋ポーリング】** 仕入先サイト選択＋#itemlist-supplierval にURL入力＋#itemlist-modal-getsupply-btn クリックを1回でまとめて実行。その後 #buyprice が埋まるまで約2秒間隔で最大10秒ポーリング（固定待ちしない）。空なら「仕入先取得フォールバック」に従う。
   - 画像入替は本ファイル「画像（写真）の入替」に従う（自動入替を確認、未入替なら #imgchange で手動入替、yui_chan_storeロゴ写真は維持）。
3. #buyshippingprice 確認（送料がかかるのに0なら岐阜県宛を入力）。#buyprice+#buyshippingprice が¥100,000以上なら自動出品中止→要承認。
4. **【まとめ実行】** #quantity=1、#customlabel 末尾に「_AI在庫補充」追記（空/プレースホルダなら短い識別子を基底に）、Shipping Policy=「No 在庫 10日　0~100ドル」（全角スペース・Copyでない先頭）、Payment=「Buy it Now 用」、Return=「free 30 days money back」、Usedなら新仕入先の状態を英訳して Condition Description 設定（Newは変更なし）を1回のJSコールでまとめて実行。#conditionid は変更しない。
5. **【利益率を12〜14%に合わせる】** #outshippingprice は最低¥3,500（未満なら3500に上書き）。#expectprofitbtn で実値を確認し、利益率が12〜14%に入るまで #price を上下して再計算（回数制限なし。12%未満→値上げ／14%超→値下げ）。
6. **【保存ガード・重要】** 保存前に①利益率が12〜14%内、②画像が新仕入先のもの（ロゴ例外時のみ既存維持）——両方を必ず確認。①未達/マイナス、または②未入替（ロゴ例外でないのに古い画像）の場合は「変更する」を押さない（①要承認/スキップ、②画像入替後に保存）。OKなら最終確認→`window.confirm=()=>true`で「変更する」保存。

### トラッカー更新（push＝Pages自動反映）
- /tmp に ebaytool を git clone --depth 1 → docs/zaiko_hoju_tracker.html の items 配列に処理行（listed/pending/skipped）を追記・更新 → commit → git push。フィールドは既存に倣う（id, category, workDate[必ず時刻まで], ebayTitle, ebayUrl, origSite, origUrl, origPrice, newSite, newUrl, newPrice, status, note）。価格ルールのJSは変更せずデータのみ。元価格不明は origPrice:null。途中終了時も処理済み分は必ずpush。承認(i)由来でlistedにしたidは localStorage の approve_<id> も削除。

### 安全・制約
- 仕入先・出品可否・利益率・画像入替は本ファイル（supplier_rules.md）を正とする。目標利益率は必ず到達・未達/マイナスは保存しない。ログイン切れ・エラーは無理に続行せず報告して終了（終了前にタブグループを閉じる）。パスワードは絶対に入力しない。
