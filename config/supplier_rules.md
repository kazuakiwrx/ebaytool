# 仕入先 共通ルール（在庫補充・セラーリサーチ 両タスク共通）

このファイルが仕入先選定・価格・利益率の唯一の基準。変更はここだけ直してpushすれば両タスクに反映される。
（このファイルは GitHub Pages の公開対象外。自動化のみが clone して読む）

## 仕入先サイト
- 使うのは **メルカリ／ヤフオク（即決のみ・オークション除外）／PayPayフリマ の3つだけ**。
- それ以外（ラクマ・楽天・Amazon・ハードオフ・駿河屋等）は使わない。
- 即決在庫のみ。複数候補は **送料込み価格の安い順** に並べ、原則1位（最安）を採用。

## 検索（仕入先の見つけ方）— recall重視（スキップを減らす）
**「見つからずスキップ」を出す前に、必ず以下を尽くすこと。** eBayタイトルをそのまま日本のフリマで検索すると当たらない（英語＋出品文句のため）。日本語＋型番で探す。
- **型番・品番を最優先**（言語非依存で最も精度が高い）。例：DW-5600STT-1JR / TD-002-J / RG 1/144 / LEGO 10233 等。型番があれば**まず型番単体で各サイト検索**。
- **日本語の正式名称に変換して検索**：英語の商品名・ブランド・キャラ名・シリーズ名・セット名を日本語へ（例 "Godzilla Soft Vinyl"→「ゴジラ ソフビ」、"Char Aznable"→「シャア・アズナブル」、"Tallgeese"→「トールギス」）。
- **eBay特有の不要語を除去**してから検索：From Japan / F/S / Free Shipping / Limited to N units / NEW / Brand New / Pre-owned / JP / Japan import / セラーの宣伝文句・数量表記など。
- **複数クエリを試す**（1つで出なくても即スキップしない）：(1)型番 (2)日本語商品名 (3)ブランド+キャラ/シリーズ (4)短いコア語のみ。語を減らして網を広げる→候補が出たら商品ページで同一性を確認。
- **3サイト（メルカリ／ヤフオク即決／PayPay）すべてで検索**。各サイト、上位だけでなくスクロールして複数件を確認。
  - メルカリ在庫あり検索URL例：https://jp.mercari.com/search?keyword=<URLエンコード>&status=on_sale
  - ヤフオクは検索後「即決（定額）」で絞る（オークションのみは対象外）。PayPayフリマも購入可能なもの。
- **売り切れ／取得失敗（買値が空）を「在庫なし」と即断しない**：同一商品の別の出品を試す、ID⇄URL再試行、別サイトへ回す（フォールバック）。
- **マッチ判定**：型番・構成・付属が一致する同一商品ならOK。コンディションはeBayのNew/Usedクラスに合わせる（細かいグレード差で弾きすぎない）。「ケースのみ/空箱/部品のみ/単品」等の明確に別物だけ除外。画像（eBay掲載写真と仕入先写真）も同一性チェックの補助に使う。
- 上記をすべて尽くしても見つからない時だけ status:"skipped"（note に試した型番/日本語キーワードを残す）。

## コンディション
- 新仕入先の商品は **元(eBay)のコンディションと同じもの** を選ぶ。
- 元の仕入先表記と eBay商品ページ(#conditionid)が食い違う場合は **eBay商品ページを正** とし、それに一致するものを選ぶ。#conditionid は変更しない。
- Used は新仕入先ページの状態（例「目立った傷や汚れなし」）を英訳して Condition Description に設定。New は商品説明変更スキップ。

## 鮮度
- **掲載6カ月以上前** の仕入先商品は対象外（各サイトの掲載日/経過表示、ヤフオクは開始日で判断）。

## 送料・原価
- **送料込みで比較**。送料が発生する場合は送料も仕入価格に含める。
- **発送先は岐阜県**。MONODAZ予想利益計算は #buyprice（商品代）+ #buyshippingprice（送料）を原価に入れる。送料が0のままで実際は送料がかかる場合は送料額を入力。

## 仕入先取得フォールバック
- 「仕入先取得」しても買値(#buyprice)が空のままなら失敗とみなす。
- (a) #itemlist-supplierval を **フルURL⇄IDに切り替えて1回再試行** → (b) それでも空なら **次に安い有効候補へ切替えて再取得**。
- 全候補で取得不可ならスキップ（理由を記録）。※PayPayフリマはフルURLで通らないことがあり、ID指定や別候補で再取得。

## 高額品
- 仕入価格（送料込み = #buyprice+#buyshippingprice）が **¥100,000以上** の商品は自動出品しない（高額のため要承認/人手確認に回す。タスク側の運用に従う）。

## 利益率・価格（タスクで異なる）
- **在庫補充（アラート版 monodaz-auto-listing ／ 出品リストQty0版 monodaz-restock-by-quantity 共通）**：出品価格は **利益率10%** になるよう調整（利益率 = 利益計 ÷ (Price × 155.8円)）。逆算（C=a·P1−profit1, a≒134.8 → P=C/(a−0.10×155.8)≒C/119.2）は初期値の目安にすぎない。**最終は必ずMONODAZの「予想利益計算」の実値で確認し、利益率が10%の±1%以内（=10%以上）になるまで価格を上下して繰り返す（回数制限なし）。利益率が目標未満／マイナスのままでは絶対に保存（変更する）しない。** 妥当な価格で10%に届かない（仕入が高すぎる等）場合は、損失を出して出品せず、見送って要承認(pending)に回すかスキップする。
- **セラーリサーチ(monodaz-hourly-10listings)**：出品価格は **「検索ebay price（リサーチ最安値）− $0.5」を基準**。最低利益率（現状5%）を下回る場合のみ満たすまで価格を上げ、下回らなければ最安値−$0.5のまま出品（無理に上げない）。

## 出品対象・除外（カテゴリ/ブランド）
- **トレーディングカード（ポケモンカード等）**：鑑定（グレーディング：PSA/BGS等）付きに限り出品OK。鑑定なしの生カードは対象外（スキップ）。eBay側・仕入先側ともに鑑定品であることを確認する。
- **コミック（本・漫画）**：出品禁止（対象外＝スキップ）。
- **Gucci／Louis Vuitton（ルイヴィトン）**：出品対象（OK）。
